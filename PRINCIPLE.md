# 🦞 龙虾 Skill 认证系统 - 原理说明

## 一句话概括

> 龙虾为你的 Skill 提供专业评估和可信签名，加密与否由你决定。

---

## 核心设计

### 1. 分离关注点

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   龙虾评估   │ ──→ │  给出建议   │ ──→ │ 开发者决策  │
│  (自动化)    │     │  (评分+理由) │     │ (加密/签名) │
└─────────────┘     └─────────────┘     └─────────────┘
```

**龙虾不做的事**:
- ❌ 强制加密
- ❌ 替你做决定
- ❌ 保存你的密码

**龙虾做的事**:
- ✅ 专业评估
- ✅ 给出建议
- ✅ 数字签名
- ✅ 区块链记录

### 2. 信任模型

```
开发者信任龙虾的评估能力
龙虾信任开发者的决策能力
区块链提供可追溯的公开证明
```

## 技术原理

### 四维评估算法

```
综合评分 = 商业价值×30% + 独特性×25% + 风险×25% + 意愿×20%
```

**为什么这样设计？**
- 商业价值最重要（30%）- 决定 skill 的市场价值
- 独特性和风险同等重要（各25%）- 决定保护必要性
- 作者意愿参考（20%）- 尊重开发者意图

### 数字签名

```
签名内容 = Skill内容哈希 + 时间戳 + 区块ID + 龙虾印记
签名格式 = YAML 嵌入 SKILL.md
验证方式 = 重新计算哈希对比
```

### 区块链

```
区块N: {
  id: N,
  skill: "skill-name",
  hash: "sha256:xxx",
  previous_hash: "区块N-1的hash",
  signature: {龙虾签名},
  evaluation: {评分详情}
}
```

**不可篡改原理**: 修改任一区块 → 哈希变化 → 后续所有区块失效

### 加密（可选）

```
加密算法: AES-256-CBC
密钥派生: PBKDF2-HMAC-SHA256 (10万次迭代)
密码: 开发者自定义（默认: xia_mi_lobster_2026）
```

## 使用方式

### 极简用法

```bash
# 看评分
python lobster_advisor.py ./my-skill

# 决定加密与否
# 加密: python encrypt_skill.py encrypt ./my-skill
# 不加密: 直接签名

# 签名
python evaluate_skill.py ./my-skill --apply
```

### 完整流程

```
创建 → 评估 → 决策 → (加密) → 签名 → 打包 → 上链
```

## 核心价值

### 对开发者

1. **专业评估** - 了解 skill 的价值和风险
2. **可信背书** - 龙虾签名增加可信度
3. **溯源保护** - 区块链记录所有权
4. **灵活选择** - 自主决定是否加密

### 对社区

1. **质量保证** - 经过评估的 skill 更可靠
2. **防抄袭** - 区块链记录原始作者
3. **透明度** - 评分标准公开
4. **可验证** - 任何人可验证签名

## 安全说明

### 加密安全
- AES-256 是军用级加密
- PBKDF2 防止暴力破解
- 盐值防止彩虹表攻击

### 签名安全
- SHA256 哈希不可逆
- 时间戳防止重放攻击
- 区块链防止篡改

### 密码安全
- 密码不存储在任何地方
- 开发者自己保管密码
- 丢失密码无法恢复（设计如此）

## 设计理念

> **"信任但验证，建议但不强制"**

龙虾相信：
- 开发者最了解自己的 skill
- 评估应该专业但建议性
- 保护应该可选而非强制
- 透明比隐藏更有价值
